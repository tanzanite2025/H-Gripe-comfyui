# H-Gripe Desktop (Tauri shell)

Phase 2 desktop shell for H-Gripe. It wraps the `hgripe-api` crate in a
[Tauri](https://tauri.app) window and exposes the API-first workflow.

## Positioning: shell panels + Node Editor, one front end

The desktop app is a **single React front end** ([`studio-ui/`](studio-ui/))
that hosts H-Gripe's own panels plus its in-house node editor as tabs. (Earlier
builds embedded ComfyUI as an "Advanced Canvas" escape hatch; that has been
removed — H-Gripe's own node graph is now the only canvas. An earlier split
between a vanilla-TS shell and an iframe-embedded editor has also been merged
into this one app.)

```
H-Gripe Desktop (single React app)
  ├─ Shell        Dashboard · Credentials · Profiles · History · PSD
  ├─ Production   PSD Studio · Run Task (H-Gripe's own panels)
  └─ Node Editor  H-Gripe's own visual workflow canvas (React Flow)
```

The app opens on **Dashboard**.

## Tabs

- **Dashboard** (default) – runtime paths + `doctor` diagnostics.
- **PSD Studio** – H-Gripe's production entry point. Compose a job from a
  provider profile + prompt + reference image + PSD template, preview the
  resulting `ApiTask`, run it through the broker, and open the outputs. The PSD
  template path is carried on the task (as `inputs.template_path`) so a future
  export step can write the generated image back into the template.
- **Credentials / Profiles** – view summaries, validate, and edit
  `credentials.json` / `provider_profiles.json` in place.
- **Run Task** – submit an `ApiTask` JSON payload to the broker and inspect the
  `ApiResult`.
- **History** – list / view / rerun / clean up recorded tasks (SQLite history).
- **PSD** – browse PSD exports (preview / metadata / smart-object markers).
- **Node Editor** – H-Gripe's own visual workflow canvas. This is the in-house
  production node graph (renderer-agnostic graph model + typed ports + DAG
  runtime), rendered as a tab in the same app (no iframe).

The whole front end is a single Vite + React + TypeScript app in
[`studio-ui/`](studio-ui/) that builds to the Tauri `frontendDist` at `dist/`
(gitignored); the Rust backend lives in `src-tauri/`. The shell panels live
under [`studio-ui/src/shell/`](studio-ui/src/shell/) and the editor under
[`studio-ui/src/editor/`](studio-ui/src/editor/); they share one i18n system
([`src/i18n.ts`](studio-ui/src/i18n.ts)) and one CSS design system
([`src/styles.css`](studio-ui/src/styles.css)). The shell-tab IPC commands go
through a single typed wrapper
([`studio-ui/src/bridge/desktop.ts`](studio-ui/src/bridge/desktop.ts)) whose
command argument / return types mirror the Rust `#[tauri::command]` signatures,
so a rename or shape change surfaces as a `tsc` error instead of a silent
runtime drift.

> **`dist/` is a build artifact.** It is generated by `studio-ui` and
> gitignored. The Tauri before* hooks build it for you; a bare `cargo run` does
> **not** run these hooks and will show a blank window, so build the front end
> first (see below) or use the Tauri CLI.

> **Security (CSP):** `tauri.conf.json` sets an explicit `app.security.csp`
> instead of `null`. It keeps `default-src 'self'` while allowing what the app
> actually needs: `style-src 'unsafe-inline'` (React / React Flow inject
> styles), `img-src data: blob:` (thumbnails are read back as data URLs), and
> `connect-src ipc: http://ipc.localhost` (Tauri IPC). The bundle is loaded as
> an external module script (`script-src 'self'`, no inline scripts). If a
> future feature needs a remote origin (web fonts, a hosted asset), widen the
> matching directive rather than reverting to `null`.

## Prerequisites (Windows)

- Rust toolchain `stable-x86_64-pc-windows-msvc`.
- Visual Studio Build Tools 2022 with the C++ workload + Windows SDK.
- WebView2 runtime (preinstalled on current Windows; otherwise install the
  Evergreen runtime).

## Build & run

```sh
# Build the front end. A plain `cargo run` does NOT build it (the window would
# be blank). The Tauri CLI runs this automatically via the before* hooks.
npm --prefix apps/desktop-tauri/studio-ui ci
npm --prefix apps/desktop-tauri/studio-ui run build   # -> apps/desktop-tauri/dist

# from the repository root (only after the front end is built)
cargo run -p hgripe-desktop          # debug run
cargo build -p hgripe-desktop --release

# or, with the Tauri CLI (npm i -g @tauri-apps/cli) — builds the front end
# for you (before* hooks)
cd apps/desktop-tauri
tauri dev                            # debug, with the frontend build hooks
tauri build
```

For front-end development you can also run the Vite dev server
(`npm --prefix apps/desktop-tauri/studio-ui run dev`) or type-check in isolation
(`npm --prefix apps/desktop-tauri/studio-ui run typecheck`).

App icons are generated from `app-icon.png` via `tauri icon app-icon.png
--output src-tauri/icons`.
